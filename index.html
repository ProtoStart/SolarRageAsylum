Solar Rage Asylum

Early taster Beta - Just launched!! 18th June 2021

New levels and game mechanics added over time

Want to join the team? Find our code on github or join the ProtoStart community subreddit

A 2D top down indy game about escaping an Asylum.

The game will be built and released incrementally over at least the remainder of 2021.
The first level will just have basic movement and very very simple to build stuff and is due out in June 2021. Once that is available we'll work on level 2. In level 2 you'll have to avoid guards, but they'll be a bit stupid and lazy - it makes it easy enough for level 2 of a game, and also means we can release level 2 without having made effective AI.

We'll add levels with new game mechanics and smarter, better equipped guards over time. By the end of 2021 there should be a pretty good game out of this - an indy game for sure, with creative game mechanics, tricky puzzles, some tense action and hopefuly an interesting story.

Oh and we'll have cheat codes and level codes, because I like old school stuff like that (and besides level codes are a quick and easy thing to get working, and don't require storing data). Try guessing some now - you might be able to get into our alpha testing area!

Currently built entirely in JavaScript, HTML and CSS. (We may add in some JS or CSS frameworks later, but for now it's all pure JS)
The game runs entirely in the players browser - which means it's not reliant on a good internet connection.

We're making our game engine from the ground up - which means it will take some time to do, but also gives us some benefits:

  • We have complete control over it - so if we want or do something very innovative or a bit out there, we can
  • It will be easy for our community to use the game engine, or bits of it, for other projects - we can even use levels of SRA to test out game mechanics that could become the core of other games
  • We can make sure that the game engine is easy to understand and work with
  • Experience: working on a game engine is a great way for developers who haven't built many games before, to really understand how games work under the hood
  • Every developer who joins up with, or collaborates with ProtoStart will be able to talk to the developers who built the game engine

Here's a little glimpse into my creative thought process:

The first level could have an intro like "We smuggled in a controller for a robot for you to aid your escape. It should be hard to detect, try moving it!" - Much better idea: you're incapacitated in the asylum, dosed up on happy pills, reliant on a (hoverwheelchair?? hoverchair?) -- hoverchair for movement. Maybe on the first level your character just wants snacks.
this gives a nice reason for some levels to have turn based blocky movement and others to have a more natural movement style when our prisoner is actually fleeing the asylum, and maybe we could have a hard to control vehicle for levels with continuos unstoppable movement.

Being dosed up on happy pills for the first few levels would be pretty cool - maybe I could make a really surreal background. It fits the first bit of my opening story draft. Having the first few levels just being life in the Asylum would also help build the narrative quite well - you're not immediatly escaping. The player gets to experience their character being in a potentially rather bleak location for a while, maybe building connections to other characters there - could we make players care about what happens to those characters??
